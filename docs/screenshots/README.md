# Screenshots

No screenshots are committed yet. This directory is reserved for real screenshots or GIFs captured from a working local or deployed demo.

Do not add placeholder image files, generated mockups, or fake Discord screenshots.

## Required Captures

Capture these after running the local demo flow:

1. `01-login-dashboard.png`: login/register screen or dashboard after sign-in.
2. `02-project-overview.png`: project overview with event volume, error rate, p95 latency, and service rollups.
3. `03-event-explorer.png`: processed events table with selected event metadata.
4. `04-anomalies.png`: anomaly table showing deterministic error-rate or latency detection.
5. `05-incident-detail.png`: incident detail with timeline, related anomalies, fingerprints, and Gemini or fallback summary.
6. `06-pipeline-health.png`: queue provider, queue depth, job counts, latency, and recent worker jobs.
7. `07-alert-history.png`: Discord alert history, or a clearly labeled skipped-alert state when `DISCORD_WEBHOOK_URL` is not configured.

## Capture Notes

- Use demo data generated by `scripts/send_demo_events.py`, `scripts/generate_error_spike.py`, and `scripts/generate_latency_spike.py`.
- Hide or crop raw API keys, provider tokens, personal email addresses, and webhook URLs.
- If a provider integration is not configured, label the fallback state honestly instead of simulating a sent alert.
- Keep image files reasonably small before committing them.
